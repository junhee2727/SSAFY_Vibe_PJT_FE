요청받은 기능을 완성하기 위한 단계별 무결성 계획을 먼저 요약합니다 — 준비 → 라우트·네비 → 캘린더 페이지 구현 → 이벤트 렌더링·상호작용 → 주변 장소 연동 → 마무리·검증. 아래는 한 치의 누락 없이 따라하면 되는 완전한 step-by-step 계획입니다.

**Phase 0 — 준비**
- **의존성 설치**: 날짜 처리를 위해 `dayjs`, 캘린더 렌더링 안정화를 위해 Vue용 FullCalendar를 권장합니다.
  - 명령(프로젝트 루트에서 실행):
    ```
    npm install dayjs @fullcalendar/vue3 @fullcalendar/daygrid @fullcalendar/interaction
    ```
  - 이유: 범위 이벤트(다중일) · 클릭/드래그 핸들링 · 이벤트 제한("더보기") 처리가 안정적으로 제공됨.
- **데이터 확인**: 행사 데이터는 `/public/json/서울_축제공연행사.json`에 있으며 `eventstartdate`, `eventenddate`, `mapx`, `mapy`, `program`, `eventplace`, `title` 등 필요한 필드가 존재합니다(예시: public/json/서울_축제공연행사.json).

**Phase 1 — 라우팅·네비게이션**
1. 라우트 추가
   - 파일: index.js
   - 작업: routes 배열에 다음 항목 추가 (lazy-load 권장):
     ```js
     { path: '/festival-calendar', name: 'FestivalCalendar', component: () => import('../pages/FestivalCalendar.vue') }
     ```
2. HeaderNav에 버튼 추가
   - 파일: HeaderNav.vue
   - 위치: `.brand` (로고) 오른쪽, 기존 `.site-nav` 왼쪽에 작은 버튼 요소 삽입.
   - 마크업 예:
     ```html
     <div class="brand-actions">
       <router-link :to="{ name: 'FestivalCalendar' }" class="calendar-btn" :class="{ active: $route.name === 'FestivalCalendar' }" aria-pressed="$route.name === 'FestivalCalendar'">
         <span class="btn-text">축제 캘린더</span>
         <span class="btn-emoji" aria-hidden>🎆</span>
       </router-link>
     </div>
     ```
   - 스타일/동작:
     - 기본: 텍스트 버튼(디자인은 기존 `.site-nav a` 스타일을 상속).
     - hover: `.btn-text`를 숨기고 `.btn-emoji`를 보이게(transition), 또는 CSS `content` 교체.
     - 클릭(포커스/active): 버튼에 `:active`로 누른 느낌 (transform: translateY(1px); box-shadow inset 등).
     - 활성화 상태: 현재 라우트가 `FestivalCalendar`이면 `active` 클래스 적용 — 시각적으로 눌러진 것처럼 보이게(기존 `router-link-active` 스타일 참고).
   - 접근성: `aria-pressed` 사용.

**Phase 2 — 페이지 및 컴포넌트 골격**
1. 새 페이지 컴포넌트 생성
   - 파일: `team-pjt-fe/src/pages/FestivalCalendar.vue` (새로 생성)
   - 구조(간단 요약):
     - `<HeaderNav />`
     - 소제목(다른 탭들처럼 동일한 스타일 사용 — 예: BoardList.vue 참조)
       - 텍스트: `축제 일정 확인해보기`
     - 캘린더 영역: `FestivalCalendarView` 컴포넌트
     - 상세 영역: `EventDetailPanel` 컴포넌트 (초기에는 빈 상태, 이벤트 클릭 시 데이터 바인딩)
2. 캘린더·상세 컴포넌트 분리
   - 파일:
     - `team-pjt-fe/src/components/FestivalCalendarView.vue` (캘린더 렌더링, 이벤트 집계/정렬/색 배정, "더보기"/주 확장 처리)
     - `team-pjt-fe/src/components/EventDetailPanel.vue` (선택 이벤트 상세, 주변 장소 목록)
   - 역할 분리로 테스트·유지보수 용이.

**Phase 3 — 데이터 로딩 및 전처리**
1. 데이터 로드 서비스 추가/확장
   - 파일: `team-pjt-fe/src/services/calendarApi.js` (새로 추가) 또는 기존 `services`에 기능 추가.
   - 기능:
     - fetch(`/json/서울_축제공연행사.json`)으로 불러오기(후보 경로 동일 로직: `/json/...`, `/static/json/...` 등).
     - `eventstartdate`/`eventenddate` 문자열을 `Date`로 변환 (예: `dayjs(eventstartdate, 'YYYYMMDD').toDate()`).
     - 각 이벤트 객체에:
       - `startDate: Date`, `endDate: Date`, `durationDays: integer` 추가.
       - 위치 좌표: `lat = Number(mapy)`, `lon = Number(mapx)` 형태로 정리.
2. 캐싱: 페이지 내에서 한 번만 로드하고 컴포넌트 상태로 재사용.

**Phase 4 — 이벤트 색상 할당 (지도 색칠 문제 해결)**
목표: 같은 날짜에 겹치는 이벤트는 서로 다른 색으로 표시, 가능한 색 수 최소화(그리디 그래프 컬러링).
1. 충돌 그래프 구성
   - 노드: 이벤트
   - 엣지: 두 이벤트의 기간이 겹치는 날짜가 하나라도 있으면 연결
2. 색상 팔레트 준비
   - 기본 팔레트: 12~16개의 식별 가능한 HSL 기반 색상(명도·채도 고정, hue 간격으로 분배).
   - 팔레트가 부족하면 추가 색은 HSL에서 hue를 더 섬세히 분배.
3. 그리디 컬러링 알고리즘
   - 이벤트를 기간 길이(또는 degree) 내림차순으로 정렬.
   - 각 이벤트에 대해 가능한 가장 낮은 색 인덱스(사용 중이지 않은) 할당.
   - 결과: 인접한 이벤트가 다른 색을 갖도록 보장(그리디는 실전에서 충분히 좋음).
4. 구현 위치: `FestivalCalendarView`에서 전처리 단계로 실행.

**Phase 5 — 캘린더 레이아웃·렌더링 규칙**
(FullCalendar 사용 시 대부분 빌트인 지원 — 아래는 FullCalendar 기반 권장 구현 + 커스텀 렌더링)
1. 월/주/일 뷰: 기본적으로 Month-Grid 뷰(꽉 찬 캘린더).
2. 이벤트 표시 규칙
   - 이벤트 텍스트: 제목만(검정 글씨).
   - 당일(eventstart == eventend): 해당 날짜의 텍스트를 bold 처리(캘린더 이벤트 텍스트 스타일).
   - 다중일 이벤트:
     - 캘린더 상에서 해당 날짜의 최상단에 색상 선(또는 스트립)으로 시작일부터 끝일까지 이어지게 표시(FullCalendar의 span 기능 또는 custom rendering로 각 셀에 배경 바 렌더).
   - 텍스트 줄임:
     - 셀 내 텍스트는 `white-space: nowrap; overflow: hidden; text-overflow: ellipsis;`
     - Hover: ellipsis 제거, `transform: translateX(0->-x)`로 왼쪽으로 천천히 스크롤 애니메이션(hover에서 `animation` 토글). 넘치는 부분은 `overflow:hidden`으로 숨김.
     - 왼쪽에 이벤트 색상 원(circle): 높이는 텍스트 높이의 80%, `border-radius:50%`, 윤곽선 없음.
3. 정렬 우선순위(같은 날짜에 여러 이벤트)
   - 1) 기간이 더 긴 이벤트 우선(기간: end-start+1)
   - 2) 기간 같으면 이름 사전순
   - 이 정렬 적용 후 표시 및 색상 배정(동일 순서가 색상 충돌 해결에도 사용됨).
4. 한 날짜에 3개까지만 표시, 4개 이상이면 3개까지 표시하고 마지막에 `+n 더보기` 버튼
   - FullCalendar의 `eventLimit`(또는 직접 렌더링 시 슬라이스 후 더보기 버튼 제공).
   - 더보기 클릭 시 해당 주(row) 높이를 확장: CSS에서 해당 주의 `.week-row` 클래스에 `height: auto`로 바꾸고 숨겨진 이벤트를 노출.
   - 더보기 확장 애니메이션: CSS transition 사용.
5. 클릭 인터랙션
   - 이벤트 클릭: `selectedEvent`로 상태 업데이트 → 아래 `EventDetailPanel`에 렌더.
   - 캘린더 상 시각적 반응:
     - 당일 이벤트: 클릭 안 해도 텍스트만 bold.
     - 다중일 이벤트: 클릭 시 해당 이벤트의 선(시작→종료) 강조(선의 불투명도/두께 증가).
   - 구현 노트: FullCalendar customRender 또는 eventClass와 external state를 사용해 선택 이벤트에 스타일 적용.

**Phase 6 — 상세 패널 및 주변 장소 목록**
1. 상세 패널(`EventDetailPanel.vue`)
   - 보여줄 항목: `title`, `eventplace`, `program`(JSON의 `program`), `tel`, `eventhomepage`, `agelimit`, `usetimefestival`, `sponsor1` 등.
   - 위치 정보가 있으면 지도 미니뷰(선택사항) 또는 링크 표시.
2. 주변 장소 계산
   - 데이터 소스: `public/json`의 다른 파일들: `서울_관광지.json`, `서울_레포츠.json`, `서울_문화시설.json`, `서울_쇼핑.json`, `서울_숙박.json`, `서울_여행코스.json` (파일명 목록은 BoardList.vue 로 확인).
   - 알고리즘:
     - 각 카테고리 파일에서 아이템의 `mapx`/`mapy`를 읽어 수치로 변환.
     - 행사 좌표와 haversine 거리 계산(단위: km): 
       ```js
       function haversine(lat1, lon1, lat2, lon2) { /* 표준 수식 */ }
       ```
     - 거리 <= 3km인 항목들을 수집해 거리순으로 정렬 후 상위 N개(예: 5개) 표시.
   - 표시: `주변 둘러볼 것` 제목 아래 리스트(카테고리, 이름, 거리, 링크).
3. 비동기 처리: 상세 패널을 렌더할 때만 주변 장소를 계산(성능 최적화).

- 클릭/포커스 접근성: keyboard focus 지원(Enter로 이벤트 선택), `aria-expanded` 등.
